Mikro’s documentation¶

Getting Started¶

Mikro (pronounced mee-cro), is a really simple game platform/engine intended for small games (you could make big ones too).

It uses Lua scripting for the games and it’s implemented in C++ for that full speed. It supports some basic shapes, sprites, animations, simple sounds and music, and everything is limited to a 320x240 screen, upscaled to a 640x480 window.

You can get Mikro here.

For you to run any game, just drag your Lua script to the executable and done. Here’s a simple “Hello World program”:

local x = 0.0
function _init()
    print("Hello World Started!")
end

function _draw()
    mik.clear(128, 80, 20)
    mik.line(0, 0, 100, 100,  255, 255, 0)
    mik.circf(x, 100, 12,  0, 255, 255)
end

function _update(dt)
    x = x + dt
end

function _exit()
    print("Hello World Finished!")
end

Now check out the Mikro’s Lua API page to learn more about all the functionality.

Mikro’s Lua API¶

The mik module wraps every functionality from Mikro’s C++ API to the Lua language.

Make sure you’ve read the Getting Started guide.

mik.quit():

Quits the game.

mik.btnp(name):

Returns true when a button has been pressed.

Parameters

name (string) – The button name. Button Constants.

mik.btnr(name):

Returns true when a button has been released.

Parameters

name (string) – The button name. Button Constants.

mik.btnh(name):

Returns true when a button has been held.

Parameters

name (string) – The button name. Button Constants.

mik.warp(x, y):

Warps the mouse to a specific location.

Parameters

  • x (number) – X coordinate.

  • y (number) – Y coordinate.

mik.mousex():

Gets the current mouse’s X coordinate.

mik.mousey():

Gets the current mouse’s Y coordinate.

mik.title([text]):

Gets/Sets the window title. If no argument is specified, it acts as a getter.

Parameters

text (string) – The text to set the window title to (optional).

Return type

string

Returns

The window title.

mik.key(r, g, b):

Sets the transparency key color.

Parameters

  • r (number) – Red component.

  • g (number) – Green component.

  • b (number) – Blue component.

mik.clip([x, y, w, h]):

Sets/Unsets the clipping region of the next drawing commands. If no argument is passed, or if any of the width and height components is zero, clipping will be disabled.

Parameters

  • x (number) – X component.

  • y (number) – Y component.

  • w (number) – Width component.

  • h (number) – Height component.

mik.clear([r, g, b]):

Clears the screen to an optional background color.

Parameters

  • r (number) – Red component.

  • g (number) – Green component.

  • b (number) – Blue component.

mik.dot(x, y, r, g, b):

Draws a dot (pixel) at a specified position (x, y) with a specified color (r, g, b).

Parameters

  • x (number) – X position component.

  • y (number) – Y position component.

  • r (number) – Red component.

  • g (number) – Green component.

  • b (number) – Blue component.

mik.line(x1, y1, x2, y2, r, g, b):

Draws a line between two point with a specified color (r, g, b).

  • x2 (number) – End X position component.

  • y2 (number) – End Y position component.

  • r (number) – Red component.

  • g (number) – Green component.

  • b (number) – Blue component.

mik.spr(sprite, x, y, [sx, sy, sw, sh, flipx, flipy]):

Draws a sprite.

Parameters

  • sprite (Sprite object) – A sprite.

  • x (number) – X component.

  • y (number) – Y component.

  • sx (number) – Source rect X component (optional).

  • sy (number) – Source rect Y component (optional).

  • sw (number) – Source rect Width component (optional).

  • sh (number) – Source rect Height component (optional).

  • flipx (bool) – If true, flips the sprite horizontally (optional).

  • flipy (bool) – If true, flips the sprite vertically (optional).

mik.pset(sprite, x, y, r, g, b, [ghost]):

Sets a pixel at a certain position in a sprite.

Parameters

  • sprite (Sprite object) – A sprite.

  • x (number) – X component.

  • y (number) – Y component.

  • r (number) – Red component.

  • g (number) – Green component.

  • b (number) – Blue component.

  • ghost (bool) – If true, the pixel will be marked as a “ghost pixel”, in other words, it will be ignored by the renderer, making it “transparent”.
